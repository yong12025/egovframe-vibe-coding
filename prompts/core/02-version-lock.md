# Prompt: Version Lock

다음 조건으로 버전 혼합 위험을 점검하고 고정 전략을 제시해줘.

입력
- 현재 의존성 목록: `<DEPENDENCY_LIST>`
- 목표 버전: `<VERSION_LANE>`
- 빌드 툴: `<BUILD_TOOL>`

작업 지시
1. 현재 의존성에서 메이저 버전 충돌 후보를 찾아.
2. 목표 버전에 맞는 수정안을 제시해.
3. 수정 후 검증 순서를 명령어 포함으로 제공해.
4. 롤백 포인트를 함께 제시해.

출력 형식
1. 충돌 후보 리스트
2. 수정안
3. 검증 명령
4. 롤백 방법
5. 리스크와 가정
