# Prompt: Scope Decomposition

요구사항을 구현 가능한 작업 단위로 분해해줘.

입력
- 버전 라인: `<VERSION_LANE>`
- 프로젝트 유형: `<STARTER_TYPE>`
- 기능 요구: `<REQUIREMENTS>`
- 제약사항: `<CONSTRAINTS>`

작업 지시
1. 기능을 도메인 단위로 분해해.
2. 공통컴포넌트 재사용 가능한 항목을 먼저 분리해.
3. 신규 구현이 필요한 항목만 백로그로 정리해.
4. 각 항목의 선행조건/완료조건을 정의해.

출력 형식
1. 분해 결과 맵
2. 재사용/신규 구현 분류표
3. 우선순위 백로그
4. 실행/검증 체크 항목
5. 리스크와 가정
