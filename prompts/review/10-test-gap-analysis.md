# Prompt: Test Gap Analysis

테스트 커버리지 갭을 분석하고 보강안을 제시해줘.

입력
- 버전 라인: `<VERSION_LANE>`
- 대상 경로: `<TARGET_PATHS>`
- 현재 테스트 목록: `<CURRENT_TESTS>`

작업 지시
1. 핵심 유스케이스 대비 테스트 누락을 식별해.
2. 단위/통합/E2E 테스트 갭을 분류해.
3. 우선순위 기반 보강 계획을 제시해.
4. 최소 릴리스 기준 테스트 세트를 제안해.

출력 형식
1. 테스트 갭 목록
2. 우선순위 계획
3. 보강 테스트 제안
4. 실행/검증 명령
5. 리스크와 가정
