# Prompt: Migration Review (v4.3 -> v5.0 beta)

아래 프로젝트의 v4.3에서 v5.0 beta 전환 계획을 수립해줘.

입력
- 현재 의존성: `<DEPENDENCY_LIST>`
- 코드 경로: `<CODE_PATHS>`
- 운영 제약: `<OPS_CONSTRAINTS>`

작업 지시
1. 호환성 리스크를 의존성/코드/API 관점으로 분류해.
2. 단계적 마이그레이션 전략을 제시해.
3. 병행 운영 기간의 안전장치를 제안해.
4. 롤백 기준과 모니터링 지표를 제시해.

출력 형식
1. 마이그레이션 리스크 맵
2. 단계별 실행 계획
3. 검증/롤백 전략
4. 일정 제안
5. 단계별 성공 기준(Go/No-Go)
6. 리스크와 가정
