# Prompt: Performance Bottleneck Review

성능 병목을 분석하고 개선안을 제시해줘.

입력
- 버전 라인: `<VERSION_LANE>`
- 대상 경로: `<TARGET_PATHS>`
- 성능 증상: `<PERF_SYMPTOMS>`
- 측정 데이터: `<METRICS_OR_PROFILE>`

작업 지시
1. 병목 후보를 애플리케이션/DB/네트워크 관점으로 분류해.
2. 병목 근거와 영향도를 제시해.
3. 단기/중기 개선안을 분리해.
4. 개선 전후 검증 방법을 제시해.

출력 형식
1. 병목 후보 목록
2. 영향도/우선순위
3. 개선안
4. 검증 명령/측정 방법
5. 리스크와 가정
