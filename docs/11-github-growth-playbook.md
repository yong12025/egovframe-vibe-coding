# GitHub Growth Playbook

이 문서는 저장소의 가시성과 신뢰도를 높여 Star/기여를 늘리기 위한 운영 가이드입니다.

---

## 1. Value Clarity

README 첫 화면에서 아래를 즉시 보여줍니다.

- 누구를 위한 저장소인지
- 어떤 문제를 해결하는지
- 5분 안에 무엇을 해볼 수 있는지

---

## 2. Fast Activation

신규 방문자가 바로 행동할 수 있게 합니다.

- 복붙 가능한 프롬프트 제공
- 예시 시나리오(`examples/`) 제공
- 기여 경로(`CONTRIBUTING.md`) 명확화
- 컨벤션 추천 자동화(`tools/convention-recommend.sh`) 제공
- 컨벤션 샘플 설정(`examples/convention-templates/`) 제공
- 컨벤션 감사 샘플 리포트(`examples/convention-audit-reports/`) 제공

---

## 3. Trust Signals

신뢰 요소를 공개합니다.

- 행동 규범: `CODE_OF_CONDUCT.md`
- 보안 정책: `SECURITY.md`
- 변경 이력: `CHANGELOG.md`
- 로드맵: `ROADMAP.md`
- 이슈/PR 템플릿: `.github/`
- 준비도 리포트/대시보드 아티팩트: `.github/workflows/standard-readiness.yml`

---

## 4. Content Cadence

아래 주기로 업데이트합니다.

- 주간: 신규 프롬프트 1개
- 격주: 실전 예시 1개
- 월간: 트러블슈팅/마이그레이션 케이스 정리

---

## 5. Star Conversion Checklist

1. README에 즉시 실행 가능한 예시가 있는가
2. 프롬프트가 목적별로 분류되어 있는가
3. 문서 탐색 경로가 단순한가
4. 신규 기여자가 30분 내 첫 PR 가능하게 설계되었는가
