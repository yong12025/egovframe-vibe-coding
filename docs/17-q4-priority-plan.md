# Q4 Priority Plan (2026)

Q4 목표 항목의 우선순위를 실행 관점으로 재정렬한 계획서입니다.

---

## 우선순위 기준

1. 선행 의존성: 다른 작업의 기반이 되는가
2. 사용자 영향도: 개발자 사용성/신뢰도 개선 효과가 큰가
3. 실행 리스크: 지연 시 전체 로드맵 영향이 큰가

---

## Priority Order

### P1. 실전 레퍼런스 아키텍처 공개

이유

- 다른 플레이북/예시/기여 가이드를 정렬하는 기준점 역할
- 신규 사용자의 이해 비용을 가장 크게 줄임

완료 기준

1. 참조 아키텍처 문서 공개
2. 최소 1개 실전 예시와 연결
3. 검증 체크리스트 포함

---

### P2. 배치/보안/운영 플레이북 고도화

이유

- 운영 안정성에 직접 영향
- P1 공개 후 세부 실행 가이드를 강화하는 단계로 적합

완료 기준

1. 배치/보안/운영 섹션별 확장 문서
2. 검증 명령/운영 체크리스트 포함
3. 케이스북/프롬프트와 상호 링크

---

### P3. 커뮤니티 기여 템플릿 개선

이유

- 중요하지만 P1/P2 대비 선행 의존성이 낮음
- Q4 후반에 반영해도 전체 흐름 영향이 상대적으로 적음

완료 기준

1. 이슈/PR 템플릿 개선안 반영
2. 기여 온보딩 시간 단축 지표 정의
3. CONTRIBUTING과 일관성 검증

---

## 실행 캘린더

- 10월: P1
- 11월: P2
- 12월: P3
