# Security Playbook (Advanced)

인증/인가/입력검증/로그보호를 운영 가능한 수준으로 강화하는 보안 플레이북입니다.

---

## 1. 목표

- 인증/인가 누락 방지
- 취약 패턴 조기 탐지
- 민감정보 노출 최소화

---

## 2. 기본 정책

1. 인증 실패(401)와 권한 실패(403) 분리
2. 입력값 검증 표준 적용
3. 민감정보 로그 마스킹
4. 보안 예외 발생 시 감사 이벤트 기록

---

## 3. 점검 프레임

### 설계 단계

- 권한 모델(역할/권한/리소스) 정의
- 보안 요구사항을 API 계약에 반영

### 구현 단계

- 인증/인가 미들웨어 적용
- 입력 검증, 에러 응답 포맷 일관화

### 운영 단계

- 보안 이벤트 모니터링
- 의심 트래픽/반복 실패 탐지

---

## 4. 검증 명령

```bash
./tools/migration-lane-check.sh <PROJECT_DIR> <v4.3|v5.0-beta>
```

연계 프롬프트

- `prompts/generators/02-login-rbac.md`
- `prompts/review/02-security-review.md`

---

## 5. 운영 체크리스트

- [ ] 401/403 응답 분기가 명확한가
- [ ] 입력 검증 누락 엔드포인트가 없는가
- [ ] 민감정보가 로그에 노출되지 않는가
- [ ] 보안 이벤트를 감사 로그로 추적 가능한가
