# Common Components Map

신규 개발 전에 반드시 확인해야 하는 공통컴포넌트 재사용 지도입니다.

---

## 핵심 원칙

- 먼저 재사용 가능성 점검
- 필요한 부분만 확장
- 대체 가능한 신규 구현 금지

---

## 기능 매핑

| 기능 | 우선 전략 | 확장 포인트 예시 |
|---|---|---|
| 로그인 | 공통컴포넌트 우선 | 인증 정책, 실패 처리 |
| 게시판 | 공통컴포넌트 우선 | 필드 확장, 화면 템플릿 |
| 사용자관리 | 공통컴포넌트 우선 | 조직/권한 모델 확장 |
| 공통코드 | 공통컴포넌트 우선 | 도메인 코드셋 추가 |
| 로그관리 | 공통컴포넌트 우선 | 감사 로그 범위 확장 |
| 배치관리 | 공통컴포넌트 우선 | 잡 정책, 재처리 전략 |

---

## 점검 질문

1. 기능의 70% 이상을 기존 컴포넌트로 충족 가능한가
2. 확장이 필요한 지점이 명확한가
3. 신규 구현 사유가 운영/보안/성능 근거를 갖는가
