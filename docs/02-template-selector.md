# Template Selector

프로젝트 시작 방식을 목적에 맞게 선택하는 가이드입니다.

---

## 선택지

### Template

일반 웹 프로젝트 시작점입니다.

추천 상황

- 홈페이지
- 포털
- 전통적 MVC 중심 서비스

---

### Boot

Spring Boot 기반 시작점입니다.

추천 상황

- 신규 개발
- 배포 자동화/운영 표준화 필요

---

### MSA

마이크로서비스 구조 시작점입니다.

추천 상황

- 도메인 분리 필요
- 팀 단위 독립 배포 필요

---

### Batch

배치/대용량 처리 중심 시작점입니다.

추천 상황

- 정기 처리
- 대량 적재/집계

---

### Mobile

모바일 연동 중심 시작점입니다.

추천 상황

- API + 모바일 앱 연계
- 인증/동기화 정책이 중요한 서비스

---

### AI

AI 기능 통합 시작점입니다.

추천 상황

- 검색 고도화
- 분류/요약/추천 기능 도입

---

## 선택 규칙

1. 버전 라인(v4.3/v5.0 beta)을 먼저 고정합니다.
2. 운영 모델(단일/분산/배치)을 기준으로 시작점을 고릅니다.
3. 공통컴포넌트 재사용 가능성을 먼저 평가합니다.
